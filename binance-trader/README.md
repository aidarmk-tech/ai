# Binance Trader — торговый бот для Android

Android-приложение, которое автоматически торгует на споте Binance по встроенной стратегии.
Работает как foreground-сервис: телефон можно заблокировать, бот продолжит следить за рынком.

## ⚠️ Важно перед использованием

- **Алгоритмическая торговля — это риск.** Никакая стратегия не гарантирует прибыль;
  можно потерять вложенные средства. Начинайте с тестнета и малых сумм.
- При создании API-ключа на Binance включите **только** права
  *Enable Reading* и *Enable Spot & Margin Trading*.
  **Никогда не включайте Enable Withdrawals** — боту вывод средств не нужен.
- Рекомендуется ограничить ключ по IP или хотя бы не публиковать его нигде.
- Ключи хранятся на телефоне в зашифрованном виде (EncryptedSharedPreferences,
  Android Keystore) и никуда, кроме api.binance.com, не отправляются.

## Как получить APK

APK собирается автоматически в GitHub Actions при каждом пуше в `binance-trader/`:

1. **Releases** → релиз **«Binance Trader (последняя сборка)»** → скачать `BinanceTrader.apk`
   (стабильная ссылка: `https://github.com/<owner>/<repo>/releases/tag/binance-trader`);
2. или вкладка **Actions** → последний запуск «Build Binance Trader APK» → артефакт.

На телефоне разрешите установку из неизвестных источников и откройте APK.

## Несколько пар одновременно

Бот следит сразу за несколькими парами и держит независимые позиции по каждой:

- **Режим AUTO** (по умолчанию): бот сам выбирает топ-8 самых ликвидных пар к USDT
  по обороту за 24 часа (стейблкоин-пары исключаются) и пересматривает список
  каждые 6 часов. Пары с открытой позицией из списка не выбрасываются, пока
  сделка не закрыта.
- **Ручной режим**: перечислите пары через запятую, например `BTCUSDT, ETHUSDT, SOLUSDT`.

Параметр **«Макс. позиций»** (по умолчанию 3) ограничивает число одновременно
открытых сделок. Параметр **«На сделку, %»** — какой процент свободного USDT
уходит в каждую сделку (проценты считаются от остатка, поэтому депозит
не может уйти в минус: 3 сделки по 20% займут 20% + 16% + 12,8%).

## Стратегия «Хамелеон»

Адаптивный ансамбль: бот сначала определяет, в каком режиме находится рынок,
и только потом решает, каким из двух движков торговать. Пробойные сигналы
в боковике и контртрендовые в тренде — главный источник убытков простых
ботов; «Хамелеон» такие сигналы просто не берёт. Лонг-онли, спот,
по каждой паре одна позиция.

### 1. Детектор режима — Kaufman Efficiency Ratio

ER за 20 свечей = |чистый ход цены| / сумма всех колебаний.
- **ER ≥ 0.25 — ТРЕНД**: цена идёт направленно → работает движок пробоя.
- **ER < 0.25 — БОКОВИК**: цена пилит на месте → работает движок выкупа паники.
- Нейтральной зоны нет: один из движков активен всегда.

### 2. Движок «Пробой» (режим ТРЕНД)

Вход, когда 15m-свеча закрылась **выше максимума последних 10 свечей**
и объём этой свечи повышенный (**z-score ≥ 0.4** к последним 20 свечам) —
движение подтверждено деньгами, а не тонким стаканом. Отсекается перегрев:
цена дальше 3.5×ATR от средней или RSI(14) ≥ 82 — вершину параболы не покупаем.

Ведение позиции: трейлинг-стоп «люстра» (максимум − 2.2×ATR), после хода
на +1 стоп-дистанцию стоп переводится **в безубыток**. Фиксированной цели
нет — тренд едем до слома (закрытие ниже SMA20 − 0.5×ATR) или до стопа.
Тайм-стоп: если через 32 свечи позиция в минусе — движение не пошло, выходим.

### 3. Движок «Выкуп паники» (режим БОКОВИК)

В боковике цена возвращается к средней. Вход, когда цена упала на
**≥1.5 сигмы ниже SMA20**, RSI(3) < 25 (краткосрочная капитуляция), и свеча
закрылась зелёной — падение уже остановилось, «нож» не ловим. Вход только
если до средней не меньше 0.5×ATR — цель должна оправдывать риск.

Выход: цель — возврат к SMA20, либо RSI(3) ≥ 60 (перепроданность снята),
либо жёсткий стоп 2×ATR, либо тайм-стоп 16 свечей — возврат к средней
либо происходит быстро, либо гипотеза неверна.

### 4. Фильтры контекста

- **Старший таймфрейм 1h**: пробои разрешены только при восходящем 1h-тренде
  (цена выше EMA50 или EMA20 растёт) — на 15m много шума, направление задаёт час.
- **Макро-фильтр BTC**: пока BTC на 1h ниже SMA200, новые входы запрещены
  по всем парам — в медвежьем рынке альты падают вместе с BTC, и лучший
  трейд — его отсутствие. Открытые позиции при этом доводятся до выхода.

### 5. Управление капиталом и дисциплина

- **Риск-паритет**: в каждой сделке рискуем **4% капитала**. Размер позиции
  вычисляется из расстояния до стопа: `объём = риск / стоп-дистанция`.
  Волатильная монета автоматически получает меньшую позицию, спокойная —
  большую, а потеря при срабатывании стопа всегда одинакова.
  Поле «На сделку, %» служит потолком размера позиции.
- **Минимальный безопасный вход = 2× биржевого минимума** (~10 USDT).
  Позиция размером с минимум (5 USDT) — ловушка: комиссия Binance берётся
  в самой монете, и после неё + небольшого падения цены остаток
  проваливается ниже минимума, из-за чего его нельзя продать — монеты
  застревают «пылью». Двойной запас гарантирует выход. Практический
  вывод: комфортный депозит для работы — от ~50 USDT (или ставьте
  «Макс. позиций» = 1 на маленьком счёте).
- **Кулдаун**: после стоп-лосса пара блокируется на 3 свечи — не «мстим» рынку.
- **Дневной лимит убытка**: минус 6% капитала за день (UTC) — новые входы
  закрыты до завтра, открытые позиции доводятся до выхода.

Ордера рыночные; количество округляется по правилам пары (LOT_SIZE / minNotional).
Сигналы по разным парам независимы; таймфрейм входа рекомендуется 15m
(контекст всегда считается по 1h).

## Использование

1. Включите **Тестнет** (по умолчанию включён), возьмите ключи на
   <https://testnet.binance.vision> (вход через GitHub) — там виртуальные деньги.
2. Введите API Key / Secret, пары (`AUTO` или список через запятую),
   процент на сделку, максимум позиций и таймфрейм.
3. Нажмите **Запустить бота**. В журнале видно каждую свечу, сигналы и сделки;
   о покупках/продажах приходят уведомления.
4. Когда убедитесь, что всё работает — выключите тестнет и введите боевые ключи.
   Приложение попросит подтвердить переход на реальные деньги.

Открытая позиция переживает перезапуск приложения и телефона: состояние (вход,
стоп, тейк) сохраняется и восстанавливается при следующем запуске бота.

## Технические детали

- Kotlin, minSdk 26 (Android 8+), без сторонних SDK — только OkHttp + Binance REST API.
- Опрос рынка каждые 20 секунд; сигналы считаются только по закрытым свечам,
  стоп/тейк проверяются по последней цене между свечами.
- Часы синхронизируются с сервером Binance (иначе подписанные запросы отклоняются).
- Сборка: `gradle assembleRelease` (CI собирает и подписывает автоматически).
