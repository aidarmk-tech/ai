# Ассоциации — Android-пасьянс по категориям

Офлайн-игра на **Kotlin + Jetpack Compose + Material 3**. Карты сгруппированы
по **категориям** (Города, Мебель, Животные, Еда), а внутри категории имеют
**ассоциативный ранг 1…6** (города — по населению, мебель/животные — по размеру,
еда — по «тяжести»). Цель — разложить все 24 карты по основаниям по возрастанию
ранга.

## Как открыть, собрать и запустить

1. Установите **Android Studio** (Hedgehog или новее).
2. `File → Open…` и выберите папку `associations/`.
3. Дождитесь синхронизации Gradle (скачаются AGP 8.5.2, Kotlin 1.9.24,
   Compose BOM 2024.09.00 и зависимости). Файл `local.properties` с путём к SDK
   Android Studio создаст автоматически.
4. Выберите эмулятор или устройство (`minSdk 24`, `targetSdk 35`) и нажмите **Run ▶**.

Сборка из терминала (если установлен Android SDK и задан `ANDROID_HOME`):

```bash
cd associations
# при первом запуске сгенерируйте wrapper:
gradle wrapper --gradle-version 8.7
./gradlew assembleDebug          # APK в app/build/outputs/apk/debug/
./gradlew installDebug           # установить на подключённое устройство
```

## Управление

- **Перетащите** карту пальцем на нужную стопку (drag-and-drop), **или**
  тапните карту и затем стопку-назначение — доступные ходы подсвечиваются.
- **Двойной тап** — авто-отправка карты на подходящее основание.
- Тап по **колоде** — добор в сброс; когда колода пуста — сброс переворачивается обратно.
- Кнопки: **Отменить** (полная история ходов), **Новая**, **Меню**.

## Правила

- На **основание** кладётся карта своей категории с рангом на 1 больше верхней
  (старт — с ранга 1).
- На **колонку** — карта с рангом на 1 **меньше** верхней и **другой** категории
  (категории чередуются). На пустую колонку — любая карта.
- Серию подходящих подряд карт можно перемещать целиком.
- Закрытая карта переворачивается, когда становится верхней.
- **Победа** — все 24 карты на основаниях.

Колода — 24 карты (4 категории × 6 рангов), поэтому раздаётся «лесенка» из
16 карт по 7 колонкам, остальные 8 — в колоду добора. Прогресс, ходы и время
сохраняются в **DataStore** и восстанавливаются при следующем запуске.

## Архитектура и развитие

- **MVVM**: единственный источник истины — `StateFlow<GameState>` в
  `GameViewModel`; UI только читает состояние и шлёт события.
- Игровая логика (`GameLogic`) — чистые функции без побочных эффектов,
  легко тестируется.
- Новая категория добавляется одним значением в `enum Category` + список
  названий в `CardContent`. Эмодзи на карте легко заменить на картинку в `CardView`.
- Заложено под расширения: уровни/наборы карт, ассоциативные цепочки,
  локализация строк, статистика.
