# Ассоциации — Android-пасьянс по категориям

Офлайн-игра на **Kotlin + Jetpack Compose + Material 3**. Каждая ассоциация —
это карта-**основа** (★, напр. «Город», «Рыцарь», «Космос») и её **предметы**
(Москва/Лондон/Париж… ; меч/щит/конь… ; ракета/звезда/комета…). Цель — собрать
предметы на свою основу.

**Правила просто:** основу кладём в свободный слот-фундамент сверху (или на
пустую колонку), а предметы — на основу своей группы в **любом порядке**.
Собранная группа уходит с поля (✓), слот освобождается. Уровень пройден, когда
собраны все группы.

**Уровни и фишки:** уровень 1 — 4 ассоциации, дальше 5, 6, 7… (до 12). За каждую
победу начисляются монеты-**фишки** 🪙 (больше за скорость и меньшее число ходов).
Уровень и фишки сохраняются между запусками. Оформление — градиенты, тени,
анимации, экран награды.

## Готовый APK (GitHub Actions)

В репозитории есть workflow `.github/workflows/build-associations.yml`, который
собирает debug-APK на раннере GitHub. После пуша в ветку:
**вкладка Actions → запуск «Build Associations APK» → Artifacts →
`associations-debug-apk`**. Установите `app-debug.apk` на телефон
(разрешив установку из неизвестных источников).

## Как открыть, собрать и запустить

1. Установите **Android Studio** (Hedgehog или новее).
2. `File → Open…` и выберите папку `associations/`.
3. Дождитесь синхронизации Gradle (скачаются AGP 8.5.2, Kotlin 1.9.24,
   Compose BOM 2024.09.00 и зависимости). Файл `local.properties` с путём к SDK
   Android Studio создаст автоматически.
4. Выберите эмулятор или устройство (`minSdk 24`, `targetSdk 35`) и нажмите **Run ▶**.

Сборка из терминала (если установлен Android SDK и задан `ANDROID_HOME`):

```bash
cd associations
./gradlew assembleDebug          # APK в app/build/outputs/apk/debug/
./gradlew installDebug           # установить на подключённое устройство
```

Gradle wrapper уже включён в репозиторий, отдельная генерация не нужна.

## Управление

- **Перетащите** карту пальцем на нужную стопку (drag-and-drop), **или**
  тапните карту и затем стопку-назначение — доступные ходы подсвечиваются.
- **Двойной тап** — авто-отправка карты на подходящее основание.
- Тап по **колоде** — добор в сброс; когда колода пуста — сброс переворачивается обратно.
- Кнопки: **Отменить** (полная история ходов), **Новая**, **Меню**.

## Правила

- **Основа** (★) — только в пустой слот-фундамент или на пустую колонку.
- **Предмет** — на фундамент своей группы (в любом порядке), на колонку с верхней
  картой той же группы, или на пустую колонку.
- Фундамент закрепляется за группой, когда на него положена основа; счётчик k/6
  показывает прогресс. Собранная группа уходит с поля.
- Закрытая карта переворачивается, когда становится верхней.
- **Победа** — собраны все группы уровня.

Прогресс, ходы, время, уровень и фишки сохраняются в **DataStore** и
восстанавливаются при следующем запуске.

## Архитектура и развитие

- **MVVM**: единственный источник истины — `StateFlow<GameState>` в
  `GameViewModel`; UI только читает состояние и шлёт события.
- Игровая логика (`GameLogic`) — чистые функции без побочных эффектов,
  легко тестируется.
- Новая категория добавляется одним значением в `enum Category` + список
  названий в `CardContent`. Эмодзи на карте легко заменить на картинку в `CardView`.
- Заложено под расширения: уровни/наборы карт, ассоциативные цепочки,
  локализация строк, статистика.
