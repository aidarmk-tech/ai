# AdShield — блокировщик рекламы (локальный VPN)

Android-приложение, которое блокирует рекламу и трекеры **без root**, поднимая
локальный VPN и фильтруя DNS-запросы. Включается/выключается одним тапом из
шторки быстрых настроек.

## Как это работает

1. Приложение поднимает `VpnService` и регистрирует виртуальный DNS-сервер
   `10.111.222.3`. Через туннель маршрутизируется **только** этот адрес — весь
   остальной трафик устройства идёт напрямую и не виден приложению.
2. Каждый DNS-запрос (UDP/53) разбирается вручную (IPv4 → UDP → DNS). Имя домена
   проверяется по чёрному списку.
3. Если домен в списке — приложение само отвечает `0.0.0.0` (реклама не
   загружается). Иначе запрос форвардится на реальный апстрим-резолвер
   (`1.1.1.1` по умолчанию) через `protect()`-сокет, минуя собственный VPN.

Подход аналогичен DNS66 / Blokada / personalDNSfilter.

## Возможности

- 🛡️ Локальный фильтрующий VPN, без root и без передачи трафика на сервера.
- ⚡ Плитка в шторке (Quick Settings) для мгновенного вкл/выкл.
- 📋 Встроенный оффлайн-список (`app/src/main/assets/hosts.txt`) — работает сразу.
- 🔄 Кнопка обновления списка из [StevenBlack/hosts](https://github.com/StevenBlack/hosts)
  (десятки тысяч доменов).
- 📊 Счётчик заблокированных запросов и foreground-уведомление с кнопкой
  «Выключить».

## Сборка

CI (`.github/workflows/build-adblock-vpn.yml`) собирает debug-APK на каждый push
в `adblock-vpn-native/**` и кладёт его в артефакты сборки (`AdShield.apk`).
На ветке `main` APK дополнительно публикуется как rolling-release
(`tag: adshield-auto`).

Локально:

```bash
cd adblock-vpn-native
gradle assembleDebug
# app/build/outputs/apk/debug/app-debug.apk
```

Сборка использует тот же keystore, что и остальной репозиторий
(`keystore/lampplayer.jks`), поэтому обновления ставятся поверх предыдущих.

## Установка плитки в шторку

После первого запуска: открыть шторку → «Изменить» (карандаш) → перетащить
плитку **AdShield** в активную зону. Первый тап откроет приложение для выдачи
разрешения на VPN; дальше плитка переключает защиту напрямую.

## Ограничения

- Фильтрация работает на уровне DNS. Если в системе включён **Private DNS
  (DNS-over-TLS)** с указанием хоста, система ходит к нему напрямую в обход
  туннеля — для работы AdShield Private DNS нужно отключить или поставить
  «Автоматически».
- Реклама, которая отдаётся с того же домена, что и контент (напр. встроенная в
  видео), DNS-фильтром не блокируется.
- `targetSdk = 33` намеренно: чтобы не возиться с обязательным
  `foregroundServiceType` на Android 14. APK при этом нормально ставится и
  работает на Android 14.
